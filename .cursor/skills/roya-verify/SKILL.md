# roya-verify

Runs the `roya` CLI to prove the map is fresh and internally consistent, and to answer graph queries.

## Locate the engine root
Run all commands from the **engine root** — the folder that contains `package.json` (with the `roya`
bin) and `engine/verify/roya.mjs`. In this repo that is `royascaffold/`. If unsure, find it:

```bash
# from the workspace root
ls royascaffold/package.json 2>/dev/null && echo "engine root = royascaffold"
```

## Commands

```bash
npm run verify        # check freshness + consistency (does NOT regenerate the index)
npm run index         # regenerate project/map/index.yaml + project/status.md
npm run status        # index then verify (use this after editing any map file)

node engine/verify/roya.mjs query     project <ID>   # a node + calls / used_by / deps / implemented_by / pending_changes
node engine/verify/roya.mjs effective project <ID>   # implemented state vs planned deltas (which change owns them)
node engine/verify/roya.mjs init      <dir>          # scaffold a new empty system
```

## Reading the report
`verify` prints `verify: PASS | STALE | DRIFT | FAIL` plus a `checks:` line. Full report at
`project/verify/verify-report.yaml`. Only **PASS** with 0 errors means the work is done. Warnings do
not fail the build but should be understood.

| check / message | meaning | fix |
|-----------------|---------|-----|
| `freshness: STALE` / `INDEX_STALE` | a map file changed but the index wasn't regenerated | `npm run index` |
| `specs: INCOMPLETE` / `SPEC_MISSING` | a `planned` (or `creates:true`) buildable node has no build `spec` | add the full `spec` (see engine `conventions.md → spec_by_kind`) |
| `BROKEN_LINK` / `BROKEN_RULE_REF` | an edge or rule id points at nothing | fix the id, or add the missing node/rule |
| `DUPLICATE_ID` | two nodes share an id | renumber one (ids are never reused) |
| `DELTA_BROKEN_TARGET` | a delta targets a node not in the map | fix the target, or set `creates: true` |
| `boundaries: FAIL` / `BOUNDARY_VIOLATION` | a cross-domain edge bypasses the public contract | route the call through the exposed contract |
| `drift: DRIFT` | `@map` code with no node, or an implemented node with no `@map` | align the annotation or the node status |
| `CROSS_PLAN_CONFLICT` (warning) | two open changes touch the same node | expected while both are open; sequence or merge one |

## Fix loop
1. `npm run status`.
2. If not PASS, fix the **first** error using the table above (errors first, then warnings that matter).
3. Re-run `npm run status`. Repeat until `verify: PASS` with 0 errors.

Never hand-edit `project/map/index.yaml` or `project/status.md` — they are generated by `roya index`.
